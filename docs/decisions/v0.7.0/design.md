# Reliable Autonomous Mode (v0.7.0) — Design Decisions

Shipped in v0.7.0. Implementation lives in `src/scripts/once/phases/`, `hooks/`, and `src/scripts/once/lock/`.

## Problem

AFK mode had no safety nets: code delivered without running tests, no crash recovery, no token/time limits, no protection against destructive git operations.

## Solution

Three feature groups — verification gates, safety hooks, and crash recovery — each implemented as small, composable modules integrated into the existing phase pipeline.

## Key Decisions

1. **Agent-based Stop hook for verification** — uses Claude Code's `type: "agent"` hook on the `Stop` event. The hook runs lint/test/typecheck and can deny the stop with `{"decision": "block", "reason": "..."}`, returning control to Claude for self-healing. Chose agent hook over command hook because verification needs to run commands AND interpret output.

2. **Self-healing retry with escalation** — attempt 1: fix specific errors. Attempt 2: revert + different approach. Attempt 3+: revert to known good. Tracked via `.clancy/verify-attempt.txt` (file-based, survives context compaction). Max retries configurable via `CLANCY_FIX_RETRIES` (default 2).

3. **Fail-open everything** — all hooks and verification gates fail-open (allow on error). Worst case = same as pre-v0.7.0. A crashed hook never blocks delivery.

4. **Lock file for crash recovery** — `.clancy/lock.json` with PID, ticket key, branch, timestamps. `isLockStale` checks PID aliveness + 24h age threshold. Resume detection checks for orphaned feature branches on stale lock cleanup.

5. **Duration-based cost estimation** — no API for actual token usage. Uses elapsed time × configurable rate (`CLANCY_TOKEN_RATE`, default 6600 tokens/min). Good enough for budget awareness.

6. **Time guard as soft warning** — integrated into the existing context-monitor PostToolUse hook (not a separate hook). Warns at 80%/100% of `CLANCY_TIME_LIMIT`. Cannot force-stop Claude — soft guidance only.

7. **Branch guard blocks patterns, not intents** — regex-based command scanning in PreToolUse. Blocks `git push --force` (but allows `--force-with-lease`), push to main/master/develop, `git reset --hard`, `git clean -f`, `git branch -D`. Disableable via `CLANCY_BRANCH_GUARD=false`.

8. **PostCompact hook re-injects ticket context** — reads `.clancy/lock.json` and outputs `additionalContext` with ticket key, title, branch, target, and description. Command-type hook (not prompt), best-effort.

9. **Session report written to `.clancy/session-report.md`** — generated by the AFK runner after loop completion. Overwrites each run (progress.txt and costs.log are append-only, so no data lost).

10. **Claude Code hook API verified** — Stop hooks support `block` decision with `reason`. `stop_hook_active` field prevents infinite loops. PostCompact injects via `additionalContext`. No `condition` syntax exists — hooks check context internally.
